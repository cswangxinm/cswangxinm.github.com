第十九篇：类高级主题

#####1. “New-Style” Class Model
Python所有的类继承自object类。所有对象都是object的实例。在3.0中所有类都是”new-style”的。对于2.x如果没有使用class(object)来定义类，也没有继承build-in类型，则认为是classic的。

#####2. Classes and types merged
在new-style类中，type和class没有区别。所以继承像list这样的type也是可以的。除了允许我们继承build-in type。对于2.x classic classes。class实例是instance，而类与类的实例不是同样的类型。

但是2.x的new-style对象类型是类。而类类型是type。(在3.0中，所有类都是这样）

#####4. Instance Slots
通过使用__slots__来限制对象可以访问到的变量。

但是你要注意下面的错误。产生该错误主要是由于有__slots__时，类不再有__dict__属性,python使用descriptors特征来管理slot属性的存储空间。

所以，如果没有__dict__，添加新的属性也就不可能了。

当然，我们可以手动包含__dict__使得两种存储方式都存在。

对于python3.0，类中的方法是simple function。不需要实例也能访问。如果方法参数中没有self，可以通过类名访问到。
